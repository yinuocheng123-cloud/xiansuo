# 仓库代理约束

## 最新一期记录
- `custom/notes/v0.8-failure-reasons-composite-indexes-and-samples.md`

## 执行准则
- 每次执行任务前先尝试同步最新仓库；若当前目录不是 Git 仓库，需明确说明原因。
- 先读历史记录，再读相关代码，再做计划，再动手修改。
- 只围绕“高定木作公开商务线索”开发，不扩展成泛行业采集或隐私信息工具。
- 坚持最小改动和最少猜测，禁止顺手做与任务无关的重构。
- 所有注释、说明文档、版本记录必须使用中文；代码命名保持英文。
- 任何有意义的实现、取舍、实验与结论，都必须记录到 `custom/notes/` 或 `custom/notes/experiments/`。

## 本仓库当前约束
- 本轮为首个 MVP 基线，暂无更早正式记录可继承。
- 采集范围仅限公开网页中的公开商务信息。
- 禁止采集私人微信、非公开手机号、登录后信息、验证码后信息和其他隐私数据。
- 评分规则、关键词规则、平台规则必须集中配置，不得散落硬编码。
